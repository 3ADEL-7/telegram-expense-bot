# Telegram Expense Bot

بوت تيليجرام بسيط لتسجيل المصاريف وعرض الإجماليات اليومية والأسبوعية والشهرية.

## المميزات

- تسجيل المصروف مع وصف اختياري.
- عرض إجمالي مصاريف اليوم.
- عرض إجمالي مصاريف هذا الأسبوع.
- عرض إجمالي مصاريف هذا الشهر.
- عرض آخر العمليات المسجلة.
- حذف عملية بالرقم.
- تخزين محلي آمن داخل ملف `JSON`.
- لا يحتاج أي مكتبات خارجية، فقط `Node.js 18+`.

## الإعداد

1. انسخ الملف `.env.example` إلى `.env`.
2. ضع توكن البوت في `BOT_TOKEN`.
3. شغّل الأمر:

```powershell
npm start
```

## الأوامر

- `/start` بدء البوت وعرض المساعدة.
- `/help` عرض التعليمات.
- `/add 25 قهوة` إضافة مصروف.
- `/today` تقرير اليوم.
- `/week` تقرير الأسبوع.
- `/month` تقرير الشهر.
- `/recent` آخر 10 عمليات.
- `/delete 3` حذف عملية برقمها.

## طريقة سريعة من داخل تيليجرام

يمكنك أيضًا استخدام الأزرار، أو إرسال رسالة بهذه الصيغة:

```text
15 غداء
```

وسيعتبرها البوت مصروفًا جديدًا إذا كنت في وضع الإضافة.

## تشغيل 24/7 على Railway

المشروع جاهز للنشر على Railway لأن فيه `package.json` مع أمر تشغيل `npm start`.

### الخطوات

1. ارفع المشروع إلى GitHub.
2. ادخل إلى Railway وأنشئ `New Project`.
3. اختر `Deploy from GitHub Repo`.
4. اختر هذا المستودع.
5. في `Variables` أضف:

```text
BOT_TOKEN=توكن_البوت
BOT_TIMEZONE=Asia/Riyadh
```

6. أضف `Volume` للبروجيكت واربطه بمسار:

```text
/app/data
```

هذا مهم لأن البوت يحفظ البيانات محليًا في `./data`، وRailway يوضح أن التطبيقات التي تكتب في مسار نسبي مثل `./data` يجب ربط الـ volume على `/app/data` حتى تبقى البيانات بعد إعادة التشغيل أو إعادة النشر.

7. تأكد أن أمر التشغيل هو:

```text
npm start
```

### ملاحظات

- إذا لم يلتقط Railway أمر التشغيل تلقائيًا، ضعه يدويًا: `npm start`.
- لا تضع ملف `.env` في GitHub إذا كان يحتوي على التوكن الحقيقي.
